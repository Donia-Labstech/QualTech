# دليل التنفيذ — نظام QualTech للري الذكي

**تطوير: مختبر الأفكار الذكية — DONIA LABS TECH**
**الموقع: Donialabstech.online**

---

## 1. نظرة عامة على النظام

QualTech هو نظام متكامل لرصد التربة (رطوبة، حرارة، ملوحة EC، حموضة pH، وعناصر NPK) والتحكم الآلي بصمام الري، مع لوحة تحكم ويب كاملة (خريطة، سجل وتقارير، محلّل ذكاء اصطناعي، وإمكانية تثبيت التطبيق على الأجهزة).

النظام يعمل في وضعين:
- **وضع المحاكاة**: لتجربة الواجهة وعرضها دون أي عتاد حقيقي (مفعّل تلقائياً).
- **وضع الجهاز الحقيقي**: للاتصال بجهاز استشعار فعلي (ESP32 / Arduino) عبر شبكة محلية أو الإنترنت.

---

## 2. محتويات الحزمة

| الملف | الوظيفة |
|---|---|
| `index.html` | الملف الرئيسي — الواجهة كاملة (تسجيل دخول، لوحة تحكم، خريطة، تقارير، إعدادات) |
| `manifest.json` | بيانات التطبيق (الاسم، الأيقونات، الألوان) المطلوبة لتفعيل "تثبيت التطبيق" |
| `service-worker.js` | يفعّل التخزين المؤقت والعمل دون اتصال، وهو شرط أساسي لقبول المتصفح تثبيت التطبيق |
| `icon-192.png` / `icon-512.png` | أيقونتا التطبيق (من شعار DONIA LABS TECH) المستخدمتان عند التثبيت |
| `favicon-48.png` | أيقونة التبويب (Favicon) |

> ⚠️ **قاعدة أساسية**: يجب أن تبقى هذه الملفات الستة **في المجلد نفسه**، بنفس الأسماء، حتى تعمل الروابط الداخلية (manifest، الأيقونات، الـ Service Worker) بشكل صحيح.

## 3. التشغيل السريع (تجربة محلية بدون استضافة)

1. ضع الملفات الستة في مجلد واحد على جهازك.
2. افتح `index.html` مباشرة بالمتصفح (نقرتين عليه).
3. سجّل الدخول بأي اسم مستخدم/كلمة مرور (نسخة تجريبية، لا تحقق فعلي من الهوية).
4. استخدم سلايدرز "وضع المحاكاة" داخل لوحة التحكم لتجربة كل السيناريوهات.

**ملاحظة:** في هذا الوضع (فتح الملف مباشرة بصيغة `file://`) **لن تعمل** ميزتا "تثبيت التطبيق" و"الاتصال بجهاز حقيقي عبر API" بسبب قيود أمنية في المتصفحات (تتطلبان استضافة حقيقية — راجع البند 4 و5).

## 4. النشر الكامل على الإنترنت (Donialabstech.online أو أي استضافة)

### 4.1 المتطلبات
- استضافة تدعم **HTTPS** (شرط إلزامي لعمل Service Worker وتثبيت التطبيق).
- أي نوع استضافة يكفي: استضافة مشتركة عادية (cPanel)، أو خدمات مجانية مثل Netlify / Vercel / GitHub Pages.

### 4.2 خطوات الرفع (مثال على استضافة عادية بـ cPanel)
1. افتح **File Manager** في لوحة تحكم الاستضافة.
2. ادخل إلى المجلد العام `public_html` (أو المجلد الفرعي للنطاق إن كان النظام على نطاق فرعي مثل `app.donialabstech.online`).
3. ارفع الملفات كما هي بدون تعديل الأسماء (الملف الرئيسي مُسمّى بالفعل `index.html` ليفتح تلقائياً عند زيارة الرابط الجذري دون كتابة اسم الملف).
4. زر الرابط من المتصفح وتأكد أن الصفحة تفتح بشكل طبيعي.

### 4.3 خطوات الرفع (مثال على Netlify — مجاني وسريع)
1. أنشئ حساباً على netlify.com.
2. اسحب وأفلت مجلد الملفات الستة في صفحة "Deploy manually".
3. سيمنحك Netlify رابطاً مباشراً يدعم HTTPS تلقائياً، يمكنك ربط نطاقكم `Donialabstech.online` به من إعدادات Domain.

### 4.4 التحقق من عمل PWA بشكل صحيح
1. افتح الموقع المنشور من Chrome على الكمبيوتر.
2. اضغط F12 لفتح أدوات المطوّر → تبويب **Application**.
3. تحقق من ظهور:
   - **Manifest**: يعرض اسم التطبيق وأيقوناته بدون أخطاء حمراء.
   - **Service Workers**: حالة "activated and is running".
4. إذا كل شيء سليم، سيظهر زر **"تثبيت التطبيق"** تلقائياً في رأس الصفحة بعد لحظات من التصفح.

## 5. ربط جهاز استشعار حقيقي (ESP32 / Arduino)

### 5.1 العتاد المطلوب (كما في النموذج التجريبي)
- لوحة تحكم ESP32 (تدعم Wi-Fi).
- مستشعر رطوبة التربة + مسبار EC/pH.
- محرك (Servo/Stepper) لتحريك المسبار رأسياً داخل التربة.
- مضخة أو صمام كهربائي للري (Relay + Solenoid Valve).
- شاشة LCD/OLED صغيرة لعرض القراءات محلياً (اختياري، موجودة في النموذج).

### 5.2 خطوات إعداد نقطة الـ API على الجهاز
1. ثبّت مكتبة `WebServer.h` (مدمجة مع حزمة ESP32 في Arduino IDE).
2. اكتب كود يقرأ المستشعرات ويرجعها بصيغة JSON عبر نقطة `/api/reading`. الكود الكامل موجود أيضاً داخل صفحة "الإعدادات" بالنظام، وملخصه:
   ```cpp
   #include <WiFi.h>
   #include <WebServer.h>
   WebServer server(80);

   void handleReading() {
     String json = "{";
     json += "\"moisture\":" + String(readMoisture()) + ",";
     json += "\"temperature\":" + String(readTemp()) + ",";
     json += "\"ec\":" + String(readEC()) + ",";
     json += "\"ph\":" + String(readPH()) + ",";
     json += "\"n\":" + String(readN()) + ",";
     json += "\"p\":" + String(readP()) + ",";
     json += "\"k\":" + String(readK()) + ",";
     json += "\"valve\":" + String(valveState ? "true" : "false");
     json += "}";
     server.send(200, "application/json", json);
   }

   void setup() {
     WiFi.begin("اسم_الشبكة", "كلمة_المرور");
     while (WiFi.status() != WL_CONNECTED) delay(300);
     server.on("/api/reading", HTTP_GET, handleReading);
     server.begin();
   }

   void loop() { server.handleClient(); }
   ```
3. بعد رفع الكود وتشغيل الجهاز، افتح **Serial Monitor** ودوّن عنوان IP الذي يطبعه الجهاز (مثل `192.168.1.50`).

### 5.3 الربط من داخل واجهة QualTech
1. سجّل الدخول إلى النظام → اذهب إلى صفحة **الإعدادات**.
2. في خانة **"رابط واجهة الجهاز (API Endpoint)"** اكتب:
   ```
   http://192.168.1.50/api/reading
   ```
3. اضغط **"اختبار الاتصال"** — يجب أن تظهر رسالة "تم الاتصال بالجهاز بنجاح".
4. فعّل مفتاح **"استخدام جهاز حقيقي (API) بدل المحاكاة"**.
5. اضغط **"حفظ الإعدادات"**. ستبدأ لوحة التحكم بقراءة البيانات الحقيقية كل 4 ثوانٍ تلقائياً.

> 💡 إذا كان الجهاز والمتصفح على شبكتين مختلفتين (مثلاً الجهاز في الحقل والمتصفح في المكتب)، يلزم نشر الجهاز عبر عنوان عام (Port Forwarding أو خدمة مثل ngrok) أو ربطه بمنصة IoT سحابية تستضيف نقطة الـ API بدلاً من الجهاز مباشرة.

## 6. ضبط حدود خوارزمية الري

في صفحة **الإعدادات → حدود خوارزمية الري**:

| الحقل | المعنى | القيمة الافتراضية |
|---|---|---|
| الحد الأدنى للرطوبة لبدء الري | إذا انخفضت الرطوبة عن هذه النسبة، يفتح الصمام تلقائياً | 20% |
| الحد الأعلى للرطوبة لإيقاف الري | إذا وصلت الرطوبة لهذه النسبة، يغلق الصمام (حماية من التشبّع) | 35% |
| أقصى ملوحة آمنة EC | إذا تجاوزت الملوحة هذا الحد، يتوقف الري وقائياً بغض النظر عن الرطوبة | 4 dS/m |
| أدنى/أعلى حموضة مقبولة pH | نطاق الحموضة المثالي المعروض في بطاقة pH | 6.0 – 7.5 |

عدّل القيم بحسب نوع المحصول الفعلي ثم اضغط **"حفظ الحدود"**.

## 7. ربط نموذج ذكاء اصطناعي خارجي (اختياري)

اللوحة الذكية تعمل افتراضياً بمحرك استدلال محلي (بدون إنترنت). لربط نموذج ذكاء اصطناعي حقيقي (مثل Claude) لتحليل أعمق:

### 7.1 لماذا لا يُستدعى النموذج مباشرة من المتصفح؟
لأن مفاتيح API يجب أن تبقى سرّية على الخادم، ولأن متصفحات الويب تمنع الاتصال المباشر بخدمات خارجية لأسباب أمنية (CORS). الحل القياسي هو إنشاء **خادم وسيط (Backend Proxy)** صغير يستقبل الطلب من الواجهة، ثم يستدعي نموذج الذكاء الاصطناعي من جهته، ويُعيد النتيجة.

### 7.2 مثال خادم وسيط بسيط (Node.js / Express)
```js
const express = require('express');
const app = express();
app.use(express.json());

app.post('/api/ai-insight', async (req, res) => {
  const { reading, thresholds } = req.body;
  try {
    const r = await fetch('https://api.anthropic.com/v1/messages', {
      method: 'POST',
      headers: {
        'x-api-key': process.env.ANTHROPIC_API_KEY, // يُحفظ في متغيرات البيئة، لا في الكود
        'anthropic-version': '2023-06-01',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        model: 'claude-sonnet-4-6',
        max_tokens: 300,
        messages: [{
          role: 'user',
          content: `حلّل قراءات التربة التالية بصيغة عملية ومختصرة بالعربية: ${JSON.stringify(reading)} مع مراعاة الحدود: ${JSON.stringify(thresholds)}`
        }]
      })
    });
    const data = await r.json();
    const advice = data.content?.[0]?.text || 'تعذّر توليد توصية الآن.';
    res.json({ advice });
  } catch (err) {
    res.status(500).json({ advice: 'حدث خطأ في الاتصال بنموذج الذكاء الاصطناعي.' });
  }
});

app.listen(3000, () => console.log('AI proxy running on port 3000'));
```

### 7.3 الربط من الواجهة
1. اذهب إلى **الإعدادات → نموذج الذكاء الاصطناعي**.
2. في خانة **"رابط نقطة نهاية التحليل"** ضع رابط خادمكم، مثل:
   ```
   https://Donialabstech.online/api/ai-insight
   ```
3. اضغط **"اختبار الاتصال"** ثم **"حفظ"**.
4. في لوحة التحكم، اضغط **"تحليل القراءة الحالية"** داخل لوحة "المحلّل الذكي للتربة" — سيظهر تحليل النموذج الحقيقي بدل التحليل المحلي.

## 9. تثبيت التطبيق على الأجهزة

| الجهاز/المتصفح | السلوك |
|---|---|
| Android (Chrome) | يظهر زر "تثبيت التطبيق" تلقائياً داخل الواجهة — اضغطه وأكّد التثبيت |
| Windows/Mac (Chrome/Edge) | نفس الزر يظهر تلقائياً، أو من أيقونة التثبيت ⊕ في شريط عنوان المتصفح |
| iPhone/iPad (Safari) | Safari لا يدعم زر التثبيت التلقائي؛ يفتح المستخدم قائمة **"مشاركة" (Share)** ثم يختار **"إضافة إلى الشاشة الرئيسية"** يدوياً |

## 10. قائمة تحقق نهائية قبل الإطلاق

- [ ] الملفات الستة مرفوعة معاً في مجلد واحد على استضافة HTTPS
- [ ] تبويب Application في أدوات المطوّر يُظهر Manifest و Service Worker بدون أخطاء
- [ ] تجربة تسجيل الدخول والتنقل بين كل الصفحات (لوحة التحكم، الخريطة، التقارير، الإعدادات)
- [ ] في حال توفر جهاز حقيقي: اختبار الاتصال بـ API ناجح، والقراءات تتحدث فعلياً كل 4 ثوانٍ
- [ ] ضبط حدود خوارزمية الري حسب نوع المحصول الفعلي
- [ ] (اختياري) ربط خادم وسيط لنموذج الذكاء الاصطناعي واختباره
- [ ] تجربة زر "تثبيت التطبيق" على جهاز Android/Windows فعلي
- [ ] تجربة زر الطباعة والتأكد من خروج تقرير نظيف
